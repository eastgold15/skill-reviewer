    📌 Three-Layer Analysis Framework  #R1-R3
    ├─ Decision: Progressive three-layer analysis (L1 → L2 → L3)
    ├─ Rationale:
    │  ├─ L1 (Engineering Correctness) is foundational
    │  ├─ L2 (Goal Achievement) is core evaluation
    │  └─ L3 (Optimization Space) is enhancement
    ├─ Implementation:
    │  ├─ L1: Check technical errors (tool calls, scripts, loops)
    │  ├─ L2: Compare design vs execution (process, output)
    │  └─ L3: Identify optimization (efficiency, tokens)
    └─ See: decisions/three-layer-analysis.md

    📌 Four Input Dimensions  #R2-R4
    ├─ Decision: 2 required + 2 optional inputs
    ├─ Required:
    │  ├─ 1. Execution Trace - What Agent actually did
    │  └─ 2. Execution Goal - What user wanted
    ├─ Optional:
    │  ├─ 3. Implementation Reference - Skill/Tool source
    │  └─ 4. Agent Implementation - Agent source (deep analysis)
    └─ Rationale: Minimum for basic analysis, more for depth

    📌 Progressive Disclosure Documentation  #R3-R4
    ├─ Decision: Keep SKILL.md concise, details in references/
    ├─ Structure:

    └─ Rationale: Quick to scan, details on-demand
